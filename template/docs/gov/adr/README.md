# adr/ — Architecture Decision Records

Why we decided what we decided.

## What is an ADR?

An Architecture Decision Record captures a significant architectural or technical decision, along with its context and consequences. ADRs are **immutable** — once accepted, they are never edited. If a decision is reversed, a new ADR is written that supersedes the old one.

## Rules

- Name format: `ADR-NNN-short-slug.md` (e.g., `ADR-001-use-postgres.md`)
- Status options: `Proposed` → `Accepted` → `Superseded by ADR-NNN`
- Never edit an accepted ADR — write a new one instead
- Reference ADRs in architecture docs and commit messages where relevant
